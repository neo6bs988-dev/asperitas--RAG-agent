# BENCHLING Benchmark Notes

Priority: P6 External Benchmark
Use: life-science data governance, registry, metadata, versioning, and audit-trail benchmark.

## Sources
1. Benchling official site
   - URL: https://www.benchling.com
2. Benchling public docs/resources
   - Use only official public materials after license/terms review.

## Patterns to Absorb
- Biological data objects need registry IDs, metadata, versioning, ownership, and audit trails.
- Workflows should preserve provenance from raw source to processed knowledge object.
- Auditability is not a later feature; it is a data-model requirement.

## Asperitas V1 Application
- Every source, chunk, retrieval result, answer, and eval result should have IDs and provenance.
- Future DBTL/protocol/sample records should inherit this audit-first schema.
- Use Benchling as a benchmark for structured R&D operations, not as a source of Asperitas facts.
