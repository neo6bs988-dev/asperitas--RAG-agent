## 5. v10.2 Universal Prompt Principles - Already Active

1. Role Prompting - assign a clear function and authority.
2. Negative Prompting - define what must not be done.
3. Context Priming - provide company, source, strategic, technical, and operational context.
4. Few-shot Prompting - provide examples of good output when useful.
5. Chain of Thought - reason internally; output only concise rationale, assumptions, checks, and conclusion.
6. Prompt Chaining - split complex work into staged outputs.
7. Meta Prompting - ask the AI to improve the prompt or process when ambiguity exists.
8. Self-Consistency - test multiple candidate answers and select the best-supported one.
9. Tree of Thought - explore major strategic branches before choosing.
10. ReAct Prompting - reason, act with tools/sources, observe, then continue.
11. Zero-shot CoT - internally reason step-by-step even when no examples exist.
12. RAG Prompting - ground answers in retrieved sources and cite/label evidence.

## 6. v10.3 Advanced Prompt Skills - Newly Added

### 1. Output Formatting
Specify the exact output shape before asking for content.  
Default: Executive Bottom Line -> Source Status -> Strategic Analysis -> Technical Skeptic Review -> Operations Plan -> Devil's Advocate -> Compliance & Trust Check -> Next Action.

### 2. Constraint Prompting
State constraints up front: budget, time, source priority, disclosure level, safety limits, no hallucination, no unverified claims, output length, format, language, decision criteria.

### 3. Persona Stacking
Use multiple expert roles in one answer. For Asperitas: CSO + Technical Skeptic + Operations Optimizer + Devil's Advocate.

### 4. Delimiter Prompting
Separate source text, task, examples, constraints, and desired output using clear delimiters:

```text
<CONTEXT>...</CONTEXT>
<TASK>...</TASK>
<SOURCES>...</SOURCES>
<CONSTRAINTS>...</CONSTRAINTS>
<OUTPUT_FORMAT>...</OUTPUT_FORMAT>
```

### 5. Iterative Refinement
Do not treat the first output as final. Produce v1 -> critique -> v2 -> final, especially for investor material, Codex prompts, grant proposals, scientific protocols, or strategic operating documents.

### 6. Assumption Surfacing
Every major output must expose assumptions. Classify them as factual, operational, technical, market, regulatory, or financial.

### 7. Perspective Shifting
Re-evaluate the same plan from CEO, COO, investor, customer, regulator, scientist, engineer, legal/compliance, and competitor perspectives.

### 8. Decomposition Prompting
Break large tasks into atomic parts: objective -> inputs -> constraints -> workflow -> owners -> metrics -> risks -> verification.

### 9. Contrastive Prompting
Compare alternatives explicitly: option A vs B vs C, strong version vs weak version, success path vs failure path, build vs buy vs partner, Korea vs U.S. vs China.

### 10. Calibration Prompting
State confidence, uncertainty, missing data, verification needs, and expected error sources. Never overstate certainty.

### 11. Schema-First Prompting
Define the data/output schema before generating content. Use tables, JSON, YAML, Markdown templates, SOP structures, or file trees.

### 12. Extended Thinking
Use deep internal reasoning for high-stakes work, but do not expose hidden chain-of-thought. Output the decision logic, assumptions, evidence, alternatives, and verification summary.

## 11. Prompt Quality Rubric

Score every important prompt from 0 to 5.

| Dimension | 0 | 5 |
|---|---|---|
| Role | No role | Clear expert/persona stack |
| Context | Missing | Company/source/task context complete |
| Task | Vague | Exact deliverable and decision objective |
| Evidence | Unsupported | Source-grounded/RAG-ready |
| Constraints | Missing | Explicit hard limits and do-not rules |
| Format | Freeform | Exact schema/table/file/output format |
| Reasoning | Shallow | Extended internal reasoning with concise rationale |
| Verification | None | Assumptions, confidence, checks, missing data |
| Iteration | One-shot | Draft -> critique -> refined final |
| Actionability | Abstract | Owner, timeline, next action, acceptance criteria |

A v10.3-grade prompt should score 42+/50.
